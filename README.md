Abstract

The name of our application is “Where To”. The first feature is “save location” which enables you to save a frequently travelled location and append it to the map to avoid a problem like repetition which can become redundant. The second feature is a “Ranking system” based on ratings for a location to allow the user to possibly choose the highest rated location of that particular search genre, this solves the problem of possibly choosing blindly or limiting options if unfamiliar with the location you’re going to. The third feature is “offline mapping” which allows you to follow a route offline without the internet as it’s common to travel in unconnected areas where mapping issues may arise. The fourth feature is “similar location” which shows locations in the same genre as the location the user is asking for, as alternatives, this is to limit the problem of restriction, the fifth feature is “”share location” which allows people on the application to share a route, this avoids problems when coordinating a trip with others where there are multiple routes that can be taken. The target audience is the general population, maps can be used by anyone regardless of age. Regarding the scope of the project, its 5 unique features/problems as opposed to an individual problem with 5 sub problems, these individual features are sufficient for 5 group members.


Problem Statement

Planning an outing, discovering new places to visit, or sharing recommendations with friends is a fragmented and frustrating experience, as users must navigate between multiple disconnected platforms to accomplish what should be a simple task. People typically use Google Maps for navigation and location information, Yelp or TripAdvisor for review and ratings, personal notes or messaging apps for saving and sharing recommendations, and social media for inspiration forcing them to switch platforms to copy and paste information, and manually coordinate them. This issue creates significant barriers: users cannot easily save and organize their favourite places in a convenient way, reviews and ratings are consistently outdated across different platforms, and sharing routes or recommendations requires tedious copy-pasting. The core problem is the absence of a single, cohesive platform that integrates every feature that was mentioned in the abstract.


How will it make life better
By combining the functionalities of many applications, travelers no longer will have to navigate a breadth of services, making what should be a carefree experience into one with unnecessary hardship. Instead, using one application, users will be able to access the functionalities of a variety of services, making for a smoother travel experience.

API’s used (plan)
This project uses Google Maps Platform API, such as the Places API, Routes API, and Geocoding API. The Geocoding API is used to convert addresses into map coordinates. The Places API then uses these coordinates to provide detailed information for precise locations. The Routes API helps calculate the most efficient path to the destination.
